OOP Concepts Cheatsheet

Learn about the core principles of Object-Oriented Programming (OOP).

Core OOP Concepts

  • Class - A blueprint for creating objects.
  • Object - An instance of a class.
  • Encapsulation - Hiding the internal state and requiring all interaction to be performed through object methods.
  • Abstraction - Hiding implementation details and exposing only essential features.
  • Inheritance - A mechanism to create a new class from an existing class.
  • Polymorphism - Ability to take many forms, enabling one interface to be used for different data types.
  • Constructor - A special method used to initialize objects.
  • Destructor - A method invoked when an object is destroyed.
  • Overloading - Defining multiple functions with the same name but different parameters.
  • Overriding - Redefining a base class method in a subclass.
  • Static Methods - Methods that belong to the class rather than an instance.
  • Interface - A contract that a class must follow.
  • Abstract Class - A class that cannot be instantiated, used to provide a base class with some common functionality.
  • Composition - Using objects of other classes as fields in a class to model real-world relationships.
  • Association - The relationship between two separate classes.
